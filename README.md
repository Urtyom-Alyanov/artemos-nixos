# Artemos NixOS configuration

Очередной flake с конфигурацией для NixOS

</div>

## В наличии

- Ядро от CachyOS для ИХРОВОЙ произовдительности
- Шумодав nnoise (`modules.audio.noise-suppression`)
- Фреймворк для модулей c автодескаверингом и DRY.
- Настроенный Language Server (nixd) под vscode,
  что позволяет работать автодополнению.
- `disko` теперь смотрит в зависимости от хоста, выделен отедльный модуль,
  чтобы его можно былоо удобнее настраивать.
- `age` файлы хранятся в `/secrets/agenix`.
- `home-manager` работает в `/home` директории и властвует там, проще говоря -
  системный конфиг и конфиг пользователя разделены.
- В качестве прокси клиента используется ядро `mihomo`, на всякий случай есть
  Throne. Конфиг для `mihomo`, кстати, хранится в `/secrets/agenix/mihomo.yaml.age`
- Для secureboot используется `lanzaboote`, который вроде улучшился с тех пор
  пока я его ставил в последний раз (год назад, полгода назад релиз полноценный
  вышел)

## Архитектура

- `/nixos` - Непосредственно конфигурации NixOS
  - `/nixos/default.nix` - Авто дискаверинг для конфигураций, лежащих в `/nixos` и не имеющих префикс `_`
  - `/nixos/_modules` - Кастомные модули
  - `/nixos/_common` - Общие настройки для хостов

## Почему решил сделать вновь эту штуку

Прошлую конфигурацию стало сложно поддерживать и развёртывать на других машина, создана она была на основе
стандартной конфигурации NixOS (без `flakes` там ещё) (поэтому там есть файл `./configuration.nix`,
хоть потом флейки и были прикручены). Развёртка системы каждый раз - был танцем с `disko` и молитвами насчёт
того, чтобы всё успешно установилось. Также хрен пойми почему всё стало компилироваться из исходников, что
меня задолбало, я же не на `Gentoo Linux` сижу, я хочу получать пакеты из бинарных кешей.

Данный flake создан полностью с нуля (то есть с помощью `nix flake init`), что далее поможет с развёрткой системы.
Для большей чистоты и простоты создания воспроизводимости было решено использовать фреймворк `flake-parts`.
